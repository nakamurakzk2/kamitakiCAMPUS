# カミタキCAMPUS デザインシステム

---

## 1. デザインの基本思想

**記憶と未来を、白黒とカラーで見せ分ける。**

- 過去や学校の記憶を語る場所は **白黒写真**
- 現在や未来、挑戦や交流を語る場所は **カラー写真**
- スクロールするほど白黒からカラーへ移行し、コンセプトが体験になる

全体のトーン: **白を基調に、余白と写真で静かな空気感をつくる。**
Web3やテクノロジー的なデザインは避け、自然・土地・記憶を感じる方向性。

---

## 2. カラー

```
メイン背景（白）      #FFFFFF
サブ背景（温白）      #F7F6F2
第三背景（薄ベージュ） #EFEDE8
見出し文字（濃）      #1A1A1A
本文文字             #3A3A38
補助文字（淡）        #8A8A84
罫線                 #E0DED8
罫線（薄）           #ECEAE4
アクセント（深緑）     #4A6B5A（hover・リンクのみ）
```

Hero / CTA の写真背景セクションのみ、白文字（#F5F3EE）を使用。
それ以外は白背景＋濃い文字で、自然で読みやすい画面を構成する。
アクセント色は最小限。基本は **白×グレー×黒のモノトーン** で成立させる。

### フォント

| 用途   | フォント                        | 意図               |

### サイズ

```
H1（Hero）    clamp(30px, 5vw, 60px)
H2（セクション） clamp(26px, 3.5vw, 42px)
Body          clamp(15px, 1.1vw, 17px)

```

見出しで記憶や余韻を出し、本文で現代性と読みやすさを担保する。

## 4. レイアウト

- 1セクション1メッセージ
- 1画面1主役写真
- テキスト幅は狭め（最大600px）
- 余白を多く取る
- カードや装飾は最小限

### 数値

```
コンテンツ最大幅    1200px
テキスト最大幅      600px
セクション上下余白   140px（モバイル: 80〜100px）
グリッド            8px単位
```

### 背景の使い分け

```
Hero        → 写真フルスクリーン（暗めオーバーレイ＋白文字）
Concept     → 白背景 #FFFFFF
Place       → 温白背景 #F7F6F2（セクション切り替えのリズム）
Community   → 白背景 #FFFFFF
CTA         → 写真フルスクリーン（暗めオーバーレイ＋白文字）
Footer      → 温白背景 #F7F6F2
```

「情報量」ではなく、**余白と写真でブランドを感じさせる**。

## 5. 写真のルール

このサイトでは、写真が文章の代わりになる。

### CSSフィルター

```css
/* 過去・記憶のセクション */
filter: grayscale(100%) contrast(1.05);

/* 現在・未来のセクション */
filter: none; /* カラーそのまま */
```

### アスペクト比ガイド

```
Hero背景        カバー（100vh）
Concept         3:4（ポートレート）
Place カード     4:5（やや縦長）
Community       4:3（横長）
CTA背景         カバー（90vh）
```

## 6. UIコンポーネント

### ボタン（4種類 — セクション背景に応じて使い分け）

**白背景セクション用:**

| タイプ       | スタイル                                       |
|-------------|-----------------------------------------------|
| Dark Filled | 背景: #1A1A1A、文字: #FFFFFF、角丸: 3px         |
| Dark Outline| 枠: #E0DED8、文字: #1A1A1A                     |

**写真背景セクション用（Hero / CTA）:**

| タイプ        | スタイル                                       |
|--------------|-----------------------------------------------|
| Light Filled | 背景: #F5F3EE、文字: #1A1A1A、角丸: 3px        |
| Light Outline| 枠: rgba(245,243,238,0.35)、文字: #F5F3EE      |

- 矢印（→）付きで方向性を示す
- hover時にわずかなtranslateY or 枠色変化
- 影は使わない

### ライン

- 1px、色: #E0DED8（白背景セクション）
- セクション区切りに `linear-gradient(to right, transparent, #ECEAE4, transparent)`

### セクションラベル

```
font-size: 11px
letter-spacing: 0.2em
text-transform: uppercase
color: #8A8A84
```

下線付き（24px幅、1px）。例: CONCEPT / PLACE / COMMUNITY

### 角丸

全体的に **3px以下**。写真は2px or なし。

## 7. ヘッダー

- 固定ヘッダー（position: fixed）
- Hero上: 透明背景＋白文字
- スクロール後: 白背景＋濃い文字＋blur＋下ボーダー
- Hero通過後: 白背景モードに自動切替

```css
/* Hero上 */
background: transparent;
color: #F5F3EE;

/* スクロール後 */
background: rgba(255, 255, 255, 0.95);
backdrop-filter: blur(20px);
border-bottom: 1px solid #ECEAE4;
color: #1A1A1A;
```

---

## 8. モーション

### スクロールアニメーション

```css
opacity: 0 → 1
translateY: 30px → 0
duration: 0.9s
easing: cubic-bezier(0.25, 0.46, 0.45, 0.94)
```

- IntersectionObserver で threshold: 0.15
- delay を 0.1s 刻みで段階的に
- 過度なアニメーションは不要
- 自動カルーセルは避ける

### ホバー

- ボタン: translateY(-1px) + 色変化
- 写真カード: scale(1.03)、duration 0.8s
- ナビリンク: 下線がwidthアニメーション

### トーン

**にぎやかさより静かな強さ。** ゆっくりフェード、わずかなスライド。

## 10. 実装補足

### 英語対応の準備

見出しやボタンが英語併記に耐えられる余白設計にしておく。
今すぐ英語ページは不要だが、後から切り替え可能な構造に。

### パフォーマンス

- フォントは `display=swap`
- 画像は適切なサイズに最適化
- スクロールアニメーションは IntersectionObserver（scroll イベントは避ける）

### アクセシビリティ

- セマンティックHTML（section, nav, header, footer）
- ボタンに適切なaria-label
- 画像に alt テキスト
- コントラスト比の確保（本文 #3A3A38 on #FFFFFF = 11.7:1）
